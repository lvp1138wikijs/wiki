Email Configuration for Mail Clients

Customers can configure their email accounts with their email client application.

They can use the following settings to configure account.

Mail Account Settings
Email Address: Username@domainname.com
Username: Username@domainname.com
Password: Email account password
SMTP/Outgoing Server: mail.domainname.com
Incoming Mail Server: mail.domainname.com
Incoming Server Type: POP/IMAP
Ports: SMTP: 25 / 26 ; POP: 110 ; IMAP: 143
Secure Ports: 995 pop3s, 465 smtps, 993 imaps

Email Infrastructure on Shared Account

In our shared account email , hosting space, and databases are hosted on different dedicated servers for redundancy. If customer did any changes/updates from client Portal, this will automatically updated in corresponding servers within few seconds/minutes.

If any of the change/update doesn't work then first verify if backend is up on the related nodes. You can check it from old ICP.

Login to Old ICP » then click on “Home” . You can see the status of server backends. Check the time Last time it checked in with the current time on db server. If you see long difference in server time with current time on db time, then the backend of that server is down. Inform Peter about it.

If a client report their emails are slow or not working. It may be due to any of the following reasons . Check the following in SERVER SIDE

Check website working or not. Just load the website in browser. If not working check it by the help of website related problems

Check Mail Server:

First confirm client domain using our mail server . You can check it from http://www.dnswatch.info/ . Put the domain and change the type to MX . Then click on resolve

Do a dig command to find the mail server

 dig domain.com MX

High server load:

Check the server load . If load is high, then alert will be received in Ticket system . Contact James immediately

Too many connections to mail server ports:

If shared server has exceeded the port limit , alert will be received and Contact James immediately

Large mail queue:

If mail queue getting high , system send alert messages . Contact James to clear it and investigate issue

IP blacklisted:

Put the shared email server IP address in http://www.mxtoolbox.com/blacklists.aspx and do a black list check . If IP listed anywhere , then contact them to remove the listing . Also check the IP address in Gmail Abuse folder. If you see any complaints on shared account, then contact client to resolve the issue as soon as possible.

problem in sending emails

The common problem with email sending is always due to the isp who block port 25, therefore we have configured port 26 for smtp server in our email server, you can ask client to use port 25 in smtp server in their email client or use their isp smtp server to send emails.

Error '550 User does not exist to you'

Bounced back email is showing error “550 User does not exist to you” This means email sending server IP do not have properly Reverse DNS setup. This is common security setup of every email server in the world, that they do not accept emails from the servers which do not have proper Reverse DNS for their IPs. You need to have your customer to contact their email server admin to setup proper rdns for their email server ip.

Ask client to provide you error message

If client is having problem and you cannot find any problem related to email address, then ask him/her to provide you the exact error message which he/she is getting from email client. Also ask them for email headers too.

Try to diagnose the problem using that error message, if you cannot then google up with this error message and find the solution.

Issues With OutLook Email Client

553 sorry, that domain isn't in my list of allowed rcpthosts

It is a silly Outlook issue that has existed for many years. Outlook maintains a nickname list that is used by both the automatic name checking and the automatic completion features. The nickname list is automatically generated while you use Outlook. If the nickname cache is corrupted, Outlook may not be able to identify recipients, may offer incorrect recipients when automatically completing the e-mail address, or may send the message to the wrong person

To resolve this issue just clear Outlook's auto completion cache:

Check the following tutorial

http://support.appriver.com/KB/a620/how-to-cleardisable-the-auto-complete-cache-in-outlook-2010.aspx [outlook 2010]
