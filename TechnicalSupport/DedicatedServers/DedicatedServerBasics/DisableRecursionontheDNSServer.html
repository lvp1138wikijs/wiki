The DNS server performs recursive queries on behalf of its DNS clients and DNS servers that have forwarded DNS client queries to it. Recursion is a name-resolution technique in which a DNS server queries other DNS servers on behalf of the requesting client to fully resolve the name and then sends an answer back to the client.

           Attackers can use recursion to deny the DNS Server service. Therefore, if a DNS server in your network is not intended to receive recursive queries, recursion should be disabled on that server.

Disable Recursion in Windows Server

Open DNS Manager.

In the console tree, right-click the applicable DNS server, then click Properties.

Where?

DNS/applicable DNS server

Click the Advanced tab.

In Server options, select the Disable recursion check box, and then click OK.

If you disable recursion on the DNS server, you will not be able to use forwarders on the same server.

Disable Recursion in Plesk

  1. Log into the Plesk Admin Panel.

Select Tools and Settings.

  1. Click DNS Template Settings from the section.
  2. Select Localnets from the DNS Recursion section.
  3. Click the OK button.

Disable Recursion in Linux

  1. Locate the BIND configuration file within the operating system. The BIND configuration file is usually located in one of the following paths:
    • /etc/bind/named.conf
    • /etc/named.conf
  2. Open the named.conf file in your preferred editor.
  3. Add the following details to the Options section:
    allow-transfer {"none";};
    allow-recursion {"none";};
    recursion no;
  4. Restart the device.
