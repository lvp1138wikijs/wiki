In order to secure your server configuration, please do the following steps:

1) Install and maintain a firewall: For securing your server you need to install and maintain a firewall in your server which manage connections to the server. In Windows, the firewall most commonly used is the Windows Advanced Firewall. In Linux, the most common firewall is iptables. Configuring iptables can be difficult. Please note that it is possible to block all incoming connections, including RDP and SSH, with your firewall.

2) Always Keep your software updated and patched:  Always try to keep 3rd party software to keep them updated and patched. Also please make sure that your WHM and Plesk installations are kept up to date.

3) Maintain regular back ups: It is useful to keep regular backups in case your server has been compromised. Both WHM and Plesk have easy-to-use backup systems to create backups of user data. If ever your server is compromised, you can simply restore your data from the earliest clean backup.

4) Change access ports: Consider changing the default ports of  your services, particularly RDP and SSH. If these services are not available on the typical ports, it becomes that much more difficult for an attacker to access your server.

5) Change passwords: Change your passwords regularly, especially if you are using an older version of Plesk or WHM. Make sure to choose a password that has mixed cases (capital letters and small letters) and numbers, preferably at least six to eight characters in length.

6) Symptoms of Server Compromise: If you feel that your server has been infected by malware, then make a complete reinstall, from the latest non-infected backup if possible.
